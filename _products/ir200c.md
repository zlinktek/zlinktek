## 产品定位

IR200C 是面向 CAN / CANFD 总线应用的工业级数据记录仪，适用于车辆、工程机械、能源设备、机器人、测试台架和工业控制系统中的总线数据记录。它延续 IR200 系列的隔离、防护和宽温设计，重点面向需要长期保存 CANFD 通信过程、复盘故障现场和验证控制逻辑的项目。

适合：

- CAN / CANFD 总线原始数据长期记录  
- 新能源、储能、BMS、电机控制器等设备调试  
- 工程机械、车辆、机器人和移动平台运行数据留痕  
- 实验台架、产线测试和耐久试验过程记录  
- 偶发通信异常、故障码触发和状态切换过程追溯  

## 核心特点

- CANFD 总线记录：面向 CAN / CANFD 通信过程进行长期数据保存  
- 高可靠本地存储：数据写入 TF 卡 / SD 卡，设备可独立运行  
- 时间戳记录：为每帧数据保留时间信息，便于后续复盘和对齐分析  
- 网络取数：支持通过以太网读取日志和进行设备维护  
- 工业级隔离防护：适合强干扰、复杂接地和长时间运行场景  
- 宽压宽温：支持 DC 5V-36V 供电与 -40℃~+85℃ 工作温度  
- 可定制解析：可按项目需求扩展 DBC 解析、字段导出或上位机分析工具  

## CAN / CANFD 总线记录

IR200C 用于记录设备运行过程中的 CAN / CANFD 通信数据。对于偶发故障、通信异常、控制策略验证和长周期耐久测试，总线记录可以提供完整的现场证据。

典型记录内容包括：

- 报文 ID  
- 数据长度  
- 数据内容  
- 时间戳  
- 通道信息  
- 错误或异常状态信息（按项目配置）  

这些数据可用于分析故障码触发前后的状态变化、控制器之间的交互顺序、关键信号的趋势变化，以及总线负载和通信稳定性。

## 面向调试与故障追溯

在 CANFD 项目中，很多问题只在特定工况、特定温度、长时间运行或瞬态负载变化时出现。IR200C 可以长期部署在设备侧，在问题出现后通过日志回放进行分析。

适用问题包括：

- 偶发故障码或保护触发  
- 控制器通信中断或超时  
- 状态机切换异常  
- 传感器数据跳变或短时丢失  
- 台架耐久试验中的长周期趋势分析  

相比临时接电脑抓包，IR200C 更适合无人值守、长时间和外场环境下的数据记录。

## 数据导出与分析

IR200C 支持将记录数据保存在本地存储介质中，并通过以太网进行读取和维护。后续可结合上位机工具进行筛选、回放、导出和分析。

可支持的交付形式包括：

- 原始帧日志  
- 按时间分段的日志文件  
- CSV 导出  
- 按项目需求扩展的信号解析结果  
- 配合 DBC 的信号级分析工具（可定制）  

这样既能保留原始总线证据，也方便研发、测试和售后人员进行更直观的数据分析。

## 工业级可靠性设计

IR200C 面向工业现场和车载/台架测试环境，重点考虑电气干扰、长时间运行和数据完整性。

可靠性设计包括：

- 接口隔离和防护设计  
- 防浪涌、防静电和 EMI 滤波  
- 宽压输入，适配车载、电池和工业电源  
- 宽温工作，适合外场和设备内部安装  
- 本地非易失存储，断开 PC 后仍可继续记录  
- 网络取数，减少拆机取卡和现场操作成本  

## 关键规格

- 产品型号：IR200C  
- 记录接口：CAN / CANFD  
- 存储介质：TF 卡 / SD 卡  
- 数据内容：时间戳、报文 ID、DLC、数据字段、通道信息等  
- 数据导出：网络读取、文件导出，可按项目需求支持 CSV  
- 管理方式：以太网取数、配置文件管理、设备状态查询  
- 供电电压：DC 5V-36V  
- 工作温度：-40℃~+85℃（不含电池）  
- 存储容量：默认 64GB，可按项目需求定制  
- 典型部署：车辆/台架旁路记录、控制器调试、长周期耐久测试

## 与其他型号的选型关系

- 如果项目是 PLC / Modbus TCP 数据采集：选择 **IR200M**  
- 如果还需要双网口串接和多设备集中监控：选择 **IR200D**  
- 如果项目是 CAN / CANFD 总线记录：选择 **IR200C**  
- 如果需要通用多接口透明记录：可考虑 **IR100 / IR200**  
- 如果涉及高保密或极端环境：可进一步评估 **CR100 / JR100**  

> 简单理解：  
> **IR200M 记录 PLC / Modbus 数据，IR200C 记录 CAN / CANFD 总线数据。**
